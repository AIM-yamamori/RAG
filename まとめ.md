# LLMとは
- 文章を理解して回答を作るAI

例:
```
質問:
    日本の首都は？
LLM:
    東京です。
```

## LLMの弱点
知らない情報には答えられない
```
質問例:
    2026年の自社輸入ルールを教えて
LLM:
    一般的な貿易知識なら答えられる
```
LLMが知らないこと
- 社内資料
- 自社PDF
- 過去帳票

など

# RAGとは
- LLMに検索能力を追加する仕組み

普通のLLM:
```
質問
 ↓
AI
 ↓
回答
```
RAG:
```
質問
 ↓
資料検索
 ↓
必要な情報取得
 ↓
AI
 ↓
回答
```

## RAGの中身
### 1. 文書を保存

例:
```
PDF
    ↓
文章化
    ↓
分割
```

### 2. 検索できる形に変換
- 文書を数字に変換する
- `Embedding` という

例：
```
輸入者
↓
[0.23,0.55,0.91]
```

### 3. 質問時に検索
```
質問：
輸入者は？
 ↓
近い文章を探す
 ↓
LLMへ渡す
```

# LM Studioとは
- 自分のPCでLLMを動かすツール

通常:
```
質問
|
クラウドAI
|
回答
```
LM Studioの場合:
```
質問
|
自分のPC
|
LLM
|
回答
```

## LM Studioの役割
RAGでは：
```
RAG:
「この情報を見つけました」
    ↓
LM Studio:
「文章にして回答します」
```

例:
```
RAG:
    参考情報:
        輸入者:
            ABC Trading
        原産国:
            China
 ↓
LM Studio:
    回答:
        輸入者はABC Tradingです。
        原産国はChinaです。
```

# Google Cloudとは
- インターネット上のコンピュータ環境

```
自分のPC：
    小さいサーバ

Google Cloud：
    大きなサーバを借りる
```

## Google Cloudで使うもの

### Cloud Storage
- ファイル置き場
```
invoice.pdf
customs.pdf
```

### Document AI
- PDFを見るAI
```
PDF
 ↓
文字・表を抽出
 ↓
データ化
```

### Vector Database
- RAG検索用
```
文章
 ↓
数字
 ↓
検索
```

# 3つの関係
- RAG
  - 必要な情報を探す仕組み
- LM Studio
  - AI回答を作るエンジン
- Google Cloud
  - データやサービスを置く場所

実際の流れの例
```
① PDFアップロード
        ↓
② Google Cloudへ保存
        ↓
③ PDF解析
        ↓
④ RAGで検索準備
        ↓
⑤ 質問
        ↓
⑥ RAGが資料検索
        ↓
⑦ LM Studioが回答生成
        ↓
⑧ ユーザー表示
```
