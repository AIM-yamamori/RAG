## 3. データベース設計

### 3.1 使用するデータベース

Cloud SQL for PostgreSQL(pgvector拡張を有効化)。インスタンスサイズはdb-f1-micro相当(学習用最小構成)とする。

### 3.2 テーブル定義:`rule_chunks`

就業規則の各チャンク(条文単位)とそのベクトルを格納するテーブル。

| カラム名 | 型 | NULL許可 | 説明 |
|---|---|---|---|
| id | SERIAL PRIMARY KEY | NOT NULL | チャンクの一意ID |
| source_file | VARCHAR(255) | NOT NULL | 元のMarkdownファイル名(例:`04_休暇制度.md`) |
| chapter_title | VARCHAR(255) | NULL可 | 章タイトル(例:`第4章 休暇制度`) |
| article_no | VARCHAR(50) | NULL可 | 条文番号(例:`第28条`) |
| article_title | VARCHAR(255) | NULL可 | 条文タイトル(例:`年次有給休暇`) |
| content | TEXT | NOT NULL | チャンクの本文テキスト |
| embedding | VECTOR(768) | NOT NULL | ベクトル埋め込み(次元数はEmbeddingモデルに依存。text-embedding-004は768次元) |
| created_at | TIMESTAMP | NOT NULL | 登録日時(デフォルト:現在時刻) |

### 3.3 インデックス

| インデックス名 | 対象カラム | 種類 | 目的 |
|---|---|---|---|
| idx_rule_chunks_embedding | embedding | IVFFlat(pgvector) | 類似度検索の高速化 |
| idx_rule_chunks_source | source_file | B-tree | ファイル名での絞り込み(任意機能用) |

※ 学習用途でデータ件数が少ない(数百チャンク程度)ため、インデックスがなくても全件スキャンで十分実用的な速度が出る想定。IVFFlatインデックスの作成は学習目的として実施する。

### 3.4 テーブル作成SQL概要

```sql
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE rule_chunks (
    id SERIAL PRIMARY KEY,
    source_file VARCHAR(255) NOT NULL,
    chapter_title VARCHAR(255),
    article_no VARCHAR(50),
    article_title VARCHAR(255),
    content TEXT NOT NULL,
    embedding VECTOR(768) NOT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT now()
);

CREATE INDEX idx_rule_chunks_embedding
    ON rule_chunks USING ivfflat (embedding vector_cosine_ops)
    WITH (lists = 10);
```

## 4. API設計(バックエンド:Cloud Run上のFastAPIアプリ)

### 4.1 エンドポイント一覧

| メソッド | パス | 概要 |
|---|---|---|
| GET | `/` | チャットUI(HTML)を返す |
| POST | `/api/chat` | 質問を受け取り、回答を生成して返す |
| GET | `/api/health` | ヘルスチェック(Cloud Run動作確認用) |

### 4.2 `POST /api/chat` 仕様

**リクエスト**

```json
{
  "question": "年次有給休暇は何日もらえますか?"
}
```

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| question | string | 必須 | ユーザーの質問文(1〜500文字程度を想定) |

**レスポンス(正常時)**

```json
{
  "answer": "勤続6か月時点で10日の年次有給休暇が付与されます。勤続年数に応じて段階的に増加し、6年6か月以上で20日となります。",
  "sources": [
    {
      "source_file": "04_休暇制度.md",
      "article_no": "第28条",
      "article_title": "年次有給休暇"
    }
  ]
}
```

| フィールド | 型 | 説明 |
|---|---|---|
| answer | string | Gemini APIが生成した回答文 |
| sources | array | 回答の根拠として使用したチャンクの出典情報(複数件) |

**レスポンス(エラー時)**

```json
{
  "error": "回答の生成中にエラーが発生しました。しばらくしてから再度お試しください。"
}
```

| HTTPステータス | 説明 |
|---|---|
| 200 | 正常終了 |
| 400 | リクエスト不正(questionが空など) |
| 500 | サーバー内部エラー(DB接続失敗、API呼び出し失敗など) |

### 4.3 `GET /api/health` 仕様

Cloud Runのヘルスチェックおよび動作確認用。DB接続確認は行わず、アプリが起動していることのみを返す簡易実装とする。

**レスポンス**

```json
{ "status": "ok" }
```

## 5. 画面設計(チャットUI)

### 5.1 画面構成

単一ページのシンプルなチャット画面とする。フレームワークは使用せず、素のHTML/CSS/JavaScriptで実装する(学習目的のため最小構成)。

```
┌──────────────────────────────────────────┐
│  就業規則チャットボット(フィクトワークス)         │
├──────────────────────────────────────────┤
│                                            │
│  [会話表示エリア]                            │
│   ユーザー: 年次有給休暇は何日もらえますか?       │
│   ボット: 勤続6か月時点で10日付与されます…        │
│           出典: 04_休暇制度.md 第28条          │
│                                            │
│   (スクロール可能)                           │
│                                            │
├──────────────────────────────────────────┤
│ [テキスト入力欄                ] [送信ボタン]   │
└──────────────────────────────────────────┘
```

### 5.2 画面項目

| 項目 | 説明 |
|---|---|
| 会話表示エリア | ユーザーの質問とボットの回答を時系列で表示。出典情報は回答の下に小さく表示 |
| テキスト入力欄 | 質問文を入力するテキストボックス |
| 送信ボタン | クリック、またはEnterキー押下で`/api/chat`へリクエスト送信 |
| ローディング表示 | 回答待機中は「回答を生成中です…」等を表示 |

### 5.3 画面遷移

単一画面のみで構成し、画面遷移は発生しない。会話履歴はブラウザのメモリ上(JavaScript変数)に保持し、リロードで消去される(要件定義書 F-07に対応)。

## 6. 外部サービス連携仕様

### 6.1 Vertex AI Embedding API

| 項目 | 内容 |
|---|---|
| 用途 | チャンク本文・質問文のベクトル化 |
| 使用モデル | text-embedding-004(または同等の最新モデル) |
| 呼び出しタイミング | (1)データ取り込みバッチ実行時(チャンクごと) (2)質問応答時(質問文ごと) |
| 認証方式 | サービスアカウント(Cloud Run実行サービスアカウントに権限付与) |

### 6.2 Gemini API

| 項目 | 内容 |
|---|---|
| 用途 | 検索されたコンテキストをもとにした回答生成 |
| 使用モデル | gemini-2.0-flash相当の軽量モデル(無料枠を考慮し高頻度利用に適したモデルを選定) |
| 呼び出しタイミング | 質問応答時(検索完了後) |
| 認証方式 | APIキー(Secret Manager経由で取得) |
| プロンプト構成 | システム指示+検索されたチャンク本文(コンテキスト)+ユーザーの質問 |

### 6.3 Secret Manager

| シークレット名 | 内容 |
|---|---|
| db-connection-string | Cloud SQLへの接続文字列(ユーザー名・パスワード・接続先を含む) |
| gemini-api-key | Gemini APIのAPIキー |

Cloud Run実行サービスアカウントに対し、Secret Manager Secret Accessorロールを付与し、起動時に環境変数として読み込む。

## 7. コスト管理設計

| 項目 | 内容 |
|---|---|
| 予算アラート | Cloud Billingで予算(例:1,000円〜2,000円程度)を設定し、50%・90%・100%到達時にメール通知する |
| Cloud SQL運用 | 検証作業を行わない時間帯はインスタンスを停止する。プロジェクト終了後はインスタンスを削除する |
| Cloud Run運用 | 最小インスタンス数を0に設定し、アクセスがない時間は課金を発生させない |
| API呼び出し管理 | データ取り込みは1回のみ実行し、再実行を避ける。動作確認の質問送信回数を記録し、無料枠の上限を意識する |
