## 1. システム概要

本システムは、就業規則(Markdownファイル)を知識源として、ユーザーの質問に対しLLMが根拠付きで回答するRAGチャットボットである。Google Cloud上に構築し、Webブラウザからチャット形式で利用する。

処理は大きく「データ取り込み(オフライン処理)」と「質問応答(オンライン処理)」の2つに分かれる。

## 3. 処理フローの概要

### 3.1 オフライン処理(データ取り込み)

開発者がローカル環境(または1回限りのバッチ実行環境)から、就業規則のMarkdownファイルをCloud SQLに取り込む。この処理は初回構築時、およびデータ更新時のみ実行する。

1. Markdownファイルを読み込む
2. 条文(章・条)単位でチャンクに分割する
3. 各チャンクをVertex AI Embedding APIでベクトル化する
4. チャンクの本文・メタデータ(ファイル名、章、条文番号)・ベクトルをCloud SQLに格納する

### 3.2 オンライン処理(質問応答)

ユーザーがWebブラウザ上のチャット画面から質問すると、以下の流れで回答が生成される。

1. ユーザーがチャット画面に質問を入力する
2. Cloud Run上のWebアプリ(バックエンド)が質問を受け取る
3. 質問文をVertex AI Embedding APIでベクトル化する
4. Cloud SQL(pgvector)に対し、類似度の高いチャンクを検索する(上位数件)
5. 検索結果(チャンク本文)と質問文を組み合わせてプロンプトを構築する
6. Gemini APIにプロンプトを送信し、回答を生成する
7. 生成された回答と出典情報をWebアプリがユーザーに返す

## 4. システム構成要素一覧

| No | 構成要素 | 役割 | 採用技術 |
|---|---|---|---|
| 1 | データ取り込みバッチ | Markdown読込・チャンク分割・ベクトル化・DB格納 | Python(ローカル実行スクリプト) |
| 2 | Embedding API | テキストをベクトルに変換 | Vertex AI(text-embedding-004 等) |
| 3 | ベクトルデータベース | チャンク・ベクトルの永続化、類似検索 | Cloud SQL for PostgreSQL + pgvector |
| 4 | バックエンドアプリ | 質問受付、検索、プロンプト構築、回答取得、API提供 | Python(FastAPI) |
| 5 | LLM(回答生成) | コンテキストを踏まえた自然文回答の生成 | Gemini API |
| 6 | フロントエンド(チャットUI) | ユーザーとの対話画面 | HTML/CSS/JavaScript(シンプルなSPA、またはバックエンドが返すHTMLテンプレート) |
| 7 | ホスティング基盤 | Webアプリの実行環境 | Cloud Run |
| 8 | 認証情報管理 | DB接続情報・APIキーの安全な管理 | Secret Manager |
| 9 | ログ・モニタリング | 動作確認、トラブルシューティング | Cloud Logging(標準出力ログを自動収集) |
| 10 | コスト管理 | 予算超過の防止 | Cloud Billing 予算アラート |

## 5. 採用方針の理由

| 検討項目 | 採用 | 理由 |
|---|---|---|
| ベクトルDB | Cloud SQL + pgvector | Vertex AI Vector Searchは常時課金が発生し学習用途にはコストリスクが高い。Cloud SQLは最小構成で低コストに抑えられ、停止・削除も容易。RDBMSの操作経験も得られる |
| LLM | Gemini API | 無料枠があり、APIキーのみで手軽に利用開始できるため学習のハードルが低い |
| アプリ提供形態 | Cloud Run | リクエストがない時間は自動的にインスタンス数0までスケールし、課金が発生しない。コンテナベースでPythonアプリをそのままデプロイできる |
| 開発言語 | Python | 要件定義書の指定どおり。Google CloudのクライアントライブラリもPython対応が充実している |

## 6. データフローの概要

```
[Markdownファイル]
      │ 読込・パース
      ▼
[チャンク(条文単位のテキスト)]
      │ ベクトル化(Embedding API)
      ▼
[チャンク本文 + メタデータ + ベクトル]
      │ INSERT
      ▼
[Cloud SQL (pgvectorテーブル)] ◀─────────────┐
      │                                      │
      │ 類似検索(コサイン類似度)               │ ベクトル化された質問
      ▼                                      │
[検索結果上位N件のチャンク] ─────┐              │
                                │ プロンプトに組込    │
                                ▼              │
                        [Gemini APIへのリクエスト]   │
                                │              │
                                ▼              │
                          [生成された回答] ──────┘ (質問はEmbedding APIにも送信される)
                                │
                                ▼
                       [ユーザーへの表示(出典付き)]
```

## 7. 環境構成

| 環境 | 用途 |
|---|---|
| ローカル開発環境 | コード作成、データ取り込みバッチの実行、簡易動作確認 |
| Google Cloud(単一プロジェクト) | Cloud SQL、Cloud Run、Secret Manager等の本番相当環境(学習用途のため開発・本番を分離しない) |

複数環境(開発/検証/本番)の分離は、学習目的かつ短期間のPoCであるため行わない。単一のGoogle Cloudプロジェクト内で完結させる。
